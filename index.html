An enquiry, verbal interest, or partial payment is not a confirmed booking. No system entry, no inventory block, no commitment at this stage. Sales Executive logs interest only.

Agent bookings (§14): Agent must be pre-registered with a signed agent agreement before any booking is accepted. Agent commission must be documented. Verbal agent assurances have no standing.
02
Ground reality disclosure
Hotels · rooms · meals · transfers · group discipline — all disclosed before any payment
Sales Executive Booking Owner (assigned)
›
14 mandatory disclosures — every customer must acknowledge each (§05):
  • Hotels may be outside city centre — airport zone, outskirts, highway-access or suburban
  • Room types: single, twin, double, triple only — no family rooms, suites, or connected rooms
  • Triple rooms may be double + extra bed or sofa bed — not 3 equal full-size beds
  • No room upgrade, view room, adjacent rooms, or same-floor rooms guaranteed
  • Some meals may be packed (early departures, long drives, flight days, remote routes)
  • Jain / special dietary must be requested in advance — last-minute not possible
  • Transfers are common group transfers — private is chargeable extra
  • Group timings are fixed — individual flexibility is limited
  • Walking is required at many European sites; coach access may be restricted
  • Early check-in / late checkout not guaranteed unless paid and confirmed in writing
  • Flights may have layovers; airline may vary from what was indicated
  • Optionals cost extra unless included in the approved package in writing
  • Special requests are not guarantees without written approval
  • Tour Manager runs the group — private customisation is not possible

Customer must sign/acknowledge each item. Booking Owner records with date. A missing acknowledgement = Sales-side failure — not Ops or Tour Manager responsibility.
03
Package & pricing locked
Departure date · package version · inclusions in writing · discount recorded
Sales Executive Branch Manager (discounts)
›
§07 Price discipline: No recorded discount = no discount. No written inclusion = no commitment. Any price deviation from standard must be approved by Branch Manager and recorded in system.

Package version locked at this stage. No amendments without the formal change-control process (§17) after this point. Agent commissions must be documented and signed.

Phase 2 Booking confirmation

04
Booking deposit collected · Booking Owner assigned
Deposit received · one named Booking Owner recorded · file opened in system
Booking Owner Finance Sales Executive
›
Booking Owner — 11 areas of duty (§03):
  • Booking accuracy — package version, pax count, price on file
  • Customer data — passport, PAN, DOB collected and verified
  • Visa readiness — docs submitted, appointment tracked, status updated
  • Payments — deposit, TCS, final balance collected on time
  • Flight sealing — name match verified, PNR confirmed
  • Rooming — single / twin / double / triple recorded
  • Meals — veg / Jain / medical dietary noted per pax
  • Special requests — approved categories only, in writing
  • Optionals sold — recorded in system with payment confirmed
  • Customer expectation — ground-reality brief on file, dated
  • Handover — clean, complete file delivered to Ops

Hand-off rule: One named owner per booking. Ownership does not transfer informally (leave, holiday). Replacement requires a documented hand-off note signed by outgoing and incoming owner. Ownership ends only at "Ops accepted" system status.
⊘
Gate B — Booking confirmation gate · 12 mandatory fields
HARD STOP ›
Owner: Booking Owner
All 12 must be complete before file status = "Confirmed":
  • Booking amount received as per policy
  • Passenger names recorded exactly as per passport
  • Passport copies collected for all passengers
  • PAN / tax details collected (where applicable)
  • Departure date selected and locked
  • Package version locked
  • Rooming type recorded per passenger
  • Payment schedule communicated in writing
  • Visa requirement explained to customer
  • Ground reality disclosure completed & signed
  • Cancellation terms shared and acknowledged
  • Discount / special price recorded (if any)

If any item missing: File status = "Incomplete Booking" — NOT "Operationally Ready".
Payment alone ≠ confirmed booking. Documentation alone ≠ confirmed booking. Verbal assurance never confirms a booking. A booking is the full set, or it is incomplete.
05
Full customer data collection
Passport · DOB · PAN · mobile · rooming · meals · emergency contact · agent details
Booking Owner
›
Mandatory at booking (§04): Full name (passport exact) · Passport copy + number · Issue & expiry date · Date of birth · Gender · PAN / tax details · Mobile & email · City of residence · Occupation · Emergency contact · Rooming type · Meal preference

Conditional: Medical / mobility (if applicable) · Special request (approved categories only, written) · Agent details (if agent booking)

Rule of thumb: Two or more "tbd" or "ask later" fields = not a confirmed booking. It is an enquiry with money attached. File must remain "Incomplete Booking" until all fields are resolved.
Phase 3 Parallel processing — Visa · Finance · Ticketing
📒 Visa
Roles
Visa Officer
Visa Team Lead Booking Owner monitors & chases
06a
Visa processing — 8 status stages
Docs collected → feasibility → appointment booked → biometrics → decision → received
Visa Officer Booking Owner (chases)
›
8 visa status stages (§08):
  • Not started — Risk. Booking Owner must chase immediately.
  • Documents pending — Booking incomplete. Cannot progress.
  • Documents received — In preparation.
  • Appointment booked — Controlled.
  • Biometrics done — Awaiting decision.
  • Visa received — Ready ✓ (file may progress)
  • Visa rejected — Retention policy applies. Escalate immediately.
  • Existing valid visa — Verify copy on file.

File cannot be "operationally ready" until status = "Visa received" or "Process active with appointment confirmed."

Late booking rule (§08): No visa feasibility confirmed → booking must not proceed for Europe. Booking Owner must escalate to management at T-45 if passport is not in hand.

Hard rule (§05): No visa feasibility confirmed = booking must not proceed for Europe. Zero exceptions.

Finance Executive Finance Manager Booking Owner (closes dues)

6 payment gates (§06) — no company action permitted before gate clears:
  • Booking Gate: Deposit received → file may be created in system
  • Visa Gate: Visa fee + package deposit → Visa team may begin processing
  • Flight Gate: Ticketing deposit received → flights may be sealed
  • Supplier Gate: Minimum collection threshold → supplier commitments may be protected
  • Final Gate: 100% package + TCS + supplements → customer may enter final handover
  • Tour Gate: Zero dues pending → final vouchers and briefing may be released

Finance records receivables. Booking Owner closes dues. Finance escalates outstanding dues at T-14 to Booking Owner and management. Exception to Final Gate requires Finance Manager + Business Head written sign-off — recorded on file.

Phase 5 Group strength & departure freeze
⚙ Operations
Roles
Ops Manager
Sales Head Management (merge / cancel decisions)
08
Group strength review
Passenger threshold assessed · go / hold / merge / cancel decision
Ops Manager Sales Head
›
§18 — Group strength decision options:
  • Proceed — minimum strength met, departure confirmed
  • Hold — extend booking window (risk accepted in writing)
  • Merge — combine with nearest departure (management sign-off required)
  • Cancel — below minimum, full refund process initiated (management sign-off required)

Sales must communicate any decision to all affected passengers immediately and in writing. No passenger should hear about a merge or cancellation from a third party.
△
Gate F — Departure freeze gate
WARNING / ESCALATION ›
Owner: Ops Manager + Sales Head + Management
Trigger: Departure below minimum strength at freeze deadline.

§20 Operational freeze — once declared: No new bookings accepted. No passenger changes without Ops written approval. Rooming, dietary, optionals and special requests finalised. Any subsequent change = formal change-control process (§17) with written approval.

Gate G — Handover acceptance gate
HARD STOP ›
Owner: Ops Controller / Ops Manager
Ops may reject handover if:
  • Any mandatory field missing from any passenger file
  • Ground reality acknowledgement not on file (dated)
  • Rooming not finalised (any TBD)
  • Outstanding dues exist without written exception approval
  • Visa status unresolved
  • Flight PNR not confirmed
  • Special request not approved in writing
  • Medical / mobility needs not documented
  • Agent booking without signed terms

Rejected handover = Sales / Booking Owner failure. Ops does not inherit an incomplete file. Booking Owner remains accountable until formal acceptance.

Escalation process — §24 · three levels
ESCALATION ›
Level 1: Booking Owner → Ops & Sales Control Desk · 24-hour resolution window
Level 2: Ops & Sales Control → Management · 4-hour escalation if unresolved
Level 3: Management decision — final · documented on file

Common escalation triggers:
  • Passport not in hand at T-45
  • Flight not sealed at T-30
  • Outstanding dues at T-14
  • Visa rejection or embassy refusal
  • Dirty file presented for handover
  • Booking Owner on leave without documented hand-off note
  • Customer complaint about undisclosed booking condition
☷
Booking Owner scorecard — §25 · seasonal review
SEASONAL REVIEW ›
Reviewed each season by Ops & Sales Control. Tracks per Booking Owner:
  • Booking errors — type and frequency
  • Late payment collections
  • Dirty handovers presented to Operations
  • Customer complaints attributable to the booking stage
  • Escalations generated
  • Gate violations (A through G) with dates
  • Incomplete files pushed forward
Persistent violations → managed performance review by management.
⊘
Non-negotiable violations — §26 · zero tolerance
ZERO TOLERANCE ›
The following are violations regardless of circumstance or explanation:
  • Unrecorded discount or special price given to customer
  • Verbal inclusion commitment not backed in writing
  • Booking marked confirmed without required payment received
  • Dirty or incomplete file pushed to Operations
  • Booking Owner responsibility transferred without documented hand-off note
  • Agent booking confirmed without signed written terms
  • No visa feasibility confirmed before accepting a late Europe booking
  • Vouchers released with outstanding dues
  • Special request guaranteed to customer without written approval on file
  • No ground reality disclosure before deposit collected

The simple rule from which everything else in this policy descends

Sell clean · Book clean · Collect on time · Freeze early · Handover complete · Operate without surprises
